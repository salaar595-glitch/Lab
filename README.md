# Lab

Цей проєкт є реалізацією дошки оголошень. Проєкт реалізований за допомогою платформи Node.js та фреймворку Express. Для зберігання даних використовується база даних SQLite.

Система дозволяє створювати, переглядати, редагувати та видаляти авторів, пости й коментарі. Дані зберігаються у трьох взаємопов'язаних таблицях бази даних. Для забезпечення цілісності даних використовуються зовнішні ключі (Foreign Keys), які встановлюють зв'язки між сутностями.

База даних складається з таблиць Users, Posts та Comments. Таблиця Users містить інформацію про авторів, таблиця Posts зберігає інформацію про пости та посилається на автора за допомогою поля authorId, а таблиця Comments містить коментарі до постів і використовує поле postId для зв'язку з таблицею Posts. Для автоматичного видалення коментарів під час видалення поста використовується каскадне видалення (ON DELETE CASCADE).

Архітектура проєкту побудована за принципом розділення відповідальності між шарами. Для роботи з базою даних використовується Repository Layer, бізнес-логіка реалізована в Service Layer, а обробка HTTP-запитів здійснюється в Controller Layer. Такий підхід спрощує підтримку та подальший розвиток застосунку.

У застосунку реалізовано повний набір CRUD-операцій для всіх сутностей. Користувач може створювати нових авторів, додавати пости та коментарі, переглядати окремі записи або списки записів, змінювати їхні дані та видаляти непотрібну інформацію. Для тестування API використовувалися HTTP-запити через браузер і Postman.

Окрім основного функціоналу, у проєкті реалізовано централізовану обробку помилок. Усі помилки повертаються клієнту у форматі JSON із відповідним кодом та повідомленням. Також додано middleware для логування запитів, який фіксує HTTP-метод, адресу запиту, код відповіді сервера та час виконання операції.

Для запуску проєкту необхідно встановити залежності командою:

npm install

Після цього сервер запускається командою:

npm start

або

npm run dev

Для створення автора використовується POST-запит до маршруту /api/users, для створення поста — /api/posts, а для створення коментаря — /api/comments. Аналогічно реалізовані маршрути для отримання, оновлення та видалення даних.
