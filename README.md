# Lab
Laboratorna robota 4
Цей проєкт є реалізацією дошки оголошень групи. Проєкт реалізований за допомогою платформи Node.js та фреймворку Express. Для зберігання даних використовується база даних SQLite. Фронтенд реалізовано на чистому HTML, CSS та JavaScript без використання фреймворків.

Система дозволяє створювати, переглядати, редагувати та видаляти пости й коментарі через зручний вебінтерфейс. Дані зберігаються у трьох взаємопов'язаних таблицях бази даних. Для забезпечення цілісності даних використовуються зовнішні ключі (Foreign Keys), які встановлюють зв'язки між сутностями.

База даних складається з таблиць Users, Posts та Comments. Таблиця Users містить інформацію про авторів, таблиця Posts зберігає інформацію про пости, а таблиця Comments містить коментарі до постів і використовує поле postId для зв'язку з таблицею Posts. Для автоматичного видалення коментарів під час видалення поста використовується каскадне видалення (ON DELETE CASCADE).

Архітектура проєкту побудована за принципом розділення відповідальності між шарами. Для роботи з базою даних використовується Repository Layer, бізнес-логіка реалізована в Service Layer, а обробка HTTP-запитів здійснюється в Controller Layer. Такий підхід спрощує підтримку та подальший розвиток застосунку.

У застосунку реалізовано повний набір CRUD-операцій для постів. Користувач може додавати пости та коментарі, переглядати списки записів, змінювати їхні дані та видаляти непотрібну інформацію. Інтерфейс реалізує чотири стани: завантаження, успіх, порожній список та помилка. Для централізованого виконання всіх HTTP-запитів з фронтенду створено окремий модуль apiClient.js з функціями getList(), getById(), create(), update(), remove().

Окрім основного функціоналу, у проєкті реалізовано централізовану обробку помилок у форматі ProblemDetails із полями status, title, detail та errors. Усі помилки повертаються клієнту у форматі JSON із відповідним кодом та повідомленням. Також додано middleware для логування запитів, який фіксує HTTP-метод, адресу запиту, код відповіді сервера та час виконання операції. На рівні сервісу реалізовано валідацію вхідних даних з перевіркою обов'язкових полів та обмежень довжини рядків.

Налаштовано політику CORS з явним переліком дозволених origins та методів, що дозволяє браузеру безпечно виконувати запити до бекенду без блокування.

Для запуску проєкту необхідно перейти у папку бекенду та встановити залежності командою:

npm install

Після цього сервер запускається командою:

npm start

або

npm run dev

Бекенд буде доступний за адресою http://localhost:3000. Для запуску фронтенду необхідно відкрити папку FrontEnd у VS Code та запустити index.html через Live Server — фронтенд буде доступний за адресою http://localhost:5500.

Для роботи з постами використовується маршрут /api/v1/posts, для коментарів — /api/v1/comments. Аналогічно реалізовані маршрути для отримання, оновлення та видалення даних.
